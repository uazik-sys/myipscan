Check Your IPv6: 12 Tests & Fixes (2025 Guide)

Updated: October 16, 2025

Short answer: IPv6 support depends on your ISP, router, and device. You can check it in seconds, and if something’s missing, enable it with a few settings changes. This guide shows how to find your IPv6 address on any device, verify end‑to‑end connectivity, and fix common issues like broken DNS, ICMPv6 blocks, and WebRTC leaks.

TL;DR — 30 seconds

  • See it: open What is My IP — if you see an address like 2a00: or 2001:, IPv6 works.
  • Test it: run test‑ipv6.com and resolve AAAA records with DNS Lookup.
  • Fix it: enable IPv6 on the router (SLAAC/DHCPv6), allow ICMPv6, and disable “Prefer IPv4” options.
  • Privacy: turn on IPv6 privacy extensions and check for WebRTC/DNS leaks.

1) Find your IPv6 address on any device

Windows

  1. Press Win+R → cmd → Enter.
  2. Run ipconfig; look for IPv6 Address under your active adapter.
  3. PowerShell alternative: Get-NetIPAddress -AddressFamily IPv6.

macOS

  1. System Settings → Network → select Wi‑Fi/Ethernet.
  2. Open Details → TCP/IP; see your IPv6 Address.
  3. Terminal: ifconfig | grep inet6 or networksetup -getinfo Wi-Fi.

Android

  1. Settings → Network & Internet → Wi‑Fi → your network.
  2. Scroll to Advanced → see IPv6 address.
  3. Note: some OEM skins hide it under “IP settings”.

iPhone / iPad (iOS)

  1. Settings → Wi‑Fi → tap “ⓘ”.
  2. Scroll to the IPv6 section.
  3. For cellular, IPv6 may be enabled but hidden; test with the tools below.

Linux

  1. Terminal: ip -6 addr show (or ifconfig).
  2. Find your active interface (e.g., wlan0 / eth0) and the inet6 line.

2) Confirm end‑to‑end IPv6 connectivity

  • Open What is My IP — it shows both IPv4 and IPv6 if available.
  • Run test‑ipv6.com to check DNS, reachability, and fallback behavior.
  • Use DNS Lookup to query AAAA records for popular domains.
  • If only IPv4 appears, your ISP or router may not provide IPv6, or the firewall blocks required traffic.

3) Enable IPv6 on your network

  1. Router: enable IPv6 (Internet settings). Choose SLAAC or DHCPv6; some ISPs require PPPoE + IPv6 or specific options.
  2. Prefix delegation: ensure you receive a /56 or /64 (varies by ISP). Enable Router Advertisements.
  3. Firewall: allow essential ICMPv6 types (Neighbor Discovery, RA, PMTU). State‑tracking firewalls handle this automatically.
  4. Devices: make sure “Prefer IPv4” or “IPv6 disabled” toggles are off.
  5. Reboot: power‑cycle router and device to pick up the new prefix.

4) Privacy & security with IPv6

  • Privacy extensions: enable temporary addresses so interface IDs rotate over time.
  • WebRTC/DNS leaks: after connecting to a VPN, test leaks via WebRTC Leak Test and DNS Lookup.
  • No NAT ≠ no security: IPv6 often works without NAT; rely on a stateful firewall to block unsolicited inbound traffic.

5) Troubleshooting guide

SymptomLikely causeFix

IPv6 address present, but sites failICMPv6 blocked; DNS brokenAllow ICMPv6; verify DNS; try different resolver (DoH/DoT)
VPN connected, IPv6 leaks real IPVPN doesn’t tunnel IPv6Use a provider with full IPv6 support or disable IPv6 while on VPN
No IPv6 on Wi‑Fi, works on EthernetAP settings or old firmwareUpdate firmware; ensure RA/PD propagate over Wi‑Fi
Only link‑local address (fe80::)No Router AdvertisementsEnable RA on router; check prefix delegation

Checklist — 12 quick tests

  1. See IPv6 on What is My IP.
  2. Run test‑ipv6.com (aim for 10/10).
  3. Resolve AAAA for a few domains via DNS Lookup.
  4. Ping an IPv6‑only site or ipv6.google.com if available.
  5. Check router WAN page: confirm IPv6 prefix delegation (/56 or /64).
  6. Ensure ICMPv6 is allowed through the firewall.
  7. On Windows/macOS, confirm IPv6 isn’t disabled or deprioritized.
  8. On VPN, verify no IPv6 leak via WebRTC test.
  9. On mobile, test both Wi‑Fi and cellular.
  10. Use traceroute6 / tracert -6 to check path.
  11. Test speed; sometimes IPv6 paths are faster.
  12. Save screenshots/results for future troubleshooting.
