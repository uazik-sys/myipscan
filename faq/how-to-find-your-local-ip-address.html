How to Find Your Local IP Address on Any Device (2025)

Updated: October 16, 2025

Local IP (a.k.a. private/LAN address) identifies your device inside the home or office network. It’s different from your public IP that websites see. Below you’ll find quick steps for Windows, macOS, Android, iPhone, Linux and a router view of all devices — plus privacy, troubleshooting, and a handy command cheat-sheet.

TL;DR — 30 seconds

  • Windows: ipconfig → “IPv4 Address”.
  • macOS/Linux: ip addr / ifconfig → look for inet.
  • Android/iOS: Wi-Fi network details → “IP address”.
  • Router: open admin panel → “Connected devices/DHCP clients”.

Tip: Your local IP will usually be from 192.168.x.x, 10.x.x.x, or 172.16–31.x.x.

Why local IP matters

  • Connect printers, NAS, smart TVs, and admin panels.
  • Forward ports or create firewall rules to a specific device.
  • Diagnose conflicts (two devices trying to use the same IP).
  • Check whether your device is on the correct subnet/VLAN.

Windows

  1. Press Win+R → type cmd → Enter.
  2. Run ipconfig. Find your active adapter (Ethernet/Wi-Fi) → IPv4 Address.
  3. PowerShell (structured output): Get-NetIPAddress -AddressFamily IPv4 | Sort-Object InterfaceAlias | Format-Table

Useful Windows commands

# Show all addresses
ipconfig /all

# Network routes (useful to spot wrong gateways)
route print

# Who has what IP/MAC on LAN (ARP cache)
arp -a

macOS

  1. System Settings → Network → choose Wi-Fi or Ethernet → see “IP Address”.
  2. Terminal alternative:
# Show addresses
ipconfig getifaddr en0      # Wi-Fi on most Macs
ipconfig getifaddr en1      # Ethernet (may vary)
ifconfig | grep "inet "     # all IPv4s

Linux

  1. Open Terminal and run:
# Addresses on all interfaces
ip -4 addr show

# Active connection only (example)
ip -4 addr show dev wlan0

# Older/compat
ifconfig | grep "inet "

Tip: The gateway is usually .1 in your subnet (e.g., 192.168.1.1). Verify with ip route.

Android

  1. Settings → Network & Internet → Wi-Fi → tap your network.
  2. “Advanced” → IP address (sometimes under “IP settings”).

Note: OEM skins place IP details differently; search “Status” or “IP”.

iPhone / iPad (iOS)

  1. Settings → Wi-Fi → tap “ⓘ” next to your network.
  2. See IP Address under the IPv4 section. (IPv6 shows separately.)

Router (see all devices)

  1. Type your router IP in a browser (often 192.168.1.1 or 192.168.0.1).
  2. Log in with admin credentials.
  3. Open Connected devices, ARP table, or DHCP clients to view hostnames, local IPs, and MAC addresses.

Local vs Public IP — what’s the difference?

Your local IP works only inside your LAN and is usually assigned by the router’s DHCP server. Your public IP is assigned by your ISP and is visible to websites/services. Check it on What is My IP.

What about local IPv6?

On dual-stack networks you may also see IPv6 addresses. Link-local (starts with fe80::) works only on the local link; global unicast looks like 2a00: or 2001:. To check IPv6 specifically and run diagnostics, see the companion guide: How to Check Your IPv6 Address.

Copy-paste command cheat-sheet

# WINDOWS
ipconfig /all
arp -a
route print

# macOS
ipconfig getifaddr en0
ifconfig | grep "inet "
netstat -rn | head

# LINUX
ip -4 addr show
ip route
arp -n

Security & privacy tips

  • Use strong Wi-Fi auth (WPA2/WPA3) + unique admin password on the router.
  • Disable or isolate guest networks; restrict LAN access if guests don’t need it.
  • Update router firmware; reboot occasionally to clear buggy DHCP states.
  • For external privacy, use a VPN to mask your public IP.
  • Create DHCP reservations for key devices (NAS/printer) to keep addresses stable.

Case example: printer offline due to IP conflict

A small office lost access to a network printer randomly. On a PC, arp -a showed two different MACs responding to 192.168.1.50. The printer and a camera were both set to the same static IP. The fix was to switch both to DHCP and then create a DHCP reservation for the printer’s MAC. The issue disappeared immediately.
