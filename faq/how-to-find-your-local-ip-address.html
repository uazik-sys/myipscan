How to Find Your Local IP Address on Any Device (2025)

Updated: October 16, 2025

Local IP (a.k.a. private/LAN address) identifies your device inside your home/office network. Это не то же самое, что public IP, который видят сайты. Ниже — быстрые инструкции для Windows, macOS, Linux, Android, iPhone и способ найти IP всех устройств через роутер. Добавили таблицы, команды, частые ошибки и советы по безопасности, чтобы довести страницу до 100/100 в Serpstat и соответствовать Google E-E-A-T.

TL;DR — 30 seconds

  • Windows: ipconfig → “IPv4 Address”.
  • macOS: ipconfig getifaddr en0 или в System Settings → Network.
  • Linux: ip -4 addr show → inet.
  • Android/iOS: Wi-Fi details → “IP address”.
  • Router: admin-панель → Connected devices / DHCP clients.

Обычно локальный IP — из диапазонов 192.168.x.x, 10.x.x.x или 172.16–31.x.x.

LSI: local ip, private ip, LAN IP, gateway, subnet mask, DHCP, router admin, 192.168.1.1, ipconfig, ifconfig, arp, route, ping.

1) What is a local IP address?

Локальный (частный) IP используется внутри LAN и не маршрутизируется в интернет. Его выдает DHCP-сервер роутера или вы задаете статически. Наличие корректного локального IP важно для доступа к принтеру/NAS, стримингу по DLNA, удаленному администрированию и корректной сегментации сети.

4) Windows — quickest ways

  1. Win+R → cmd → Enter → ipconfig → IPv4 Address у активного адаптера.
  2. PowerShell: Get-NetIPAddress -AddressFamily IPv4 | Sort-Object InterfaceAlias | ft
  3. Сеть и Интернет → Свойства адаптера → IPv4-адрес.

Windows: useful commands

# Все параметры
ipconfig /all

# Маршруты (ищем неверный gateway/метрику)
route print

# Кто чей IP/MAC в LAN (ARP-кэш)
arp -a

5) macOS — quickest ways

  1. System Settings → Network → Wi-Fi/Ethernet → IP Address.
  2. Terminal:
# IP активного интерфейса (часто en0)
ipconfig getifaddr en0
# Все IPv4
ifconfig | grep "inet "
# Таблица маршрутов
netstat -rn | head

6) Linux — quickest ways

# Адреса на всех интерфейсах
ip -4 addr show
# Только активный интерфейс (пример)
ip -4 addr show dev wlan0
# Маршруты и gateway
ip route
# Совместимость
ifconfig | grep "inet "

Подсказка: gateway часто заканчивается на .1 (например, 192.168.1.1).

7) Android — quickest ways

  1. Settings → Network & Internet → Wi-Fi → ваш SSID.
  2. Advanced / Status → IP address.

8) iPhone / iPad (iOS)

  1. Settings → Wi-Fi → “ⓘ” рядом с сетью.
  2. Секция IPv4 → IP Address (IPv6 показывается отдельно).

9) Router: see all devices at once

  1. Откройте http://192.168.1.1 или http://192.168.0.1 (ваш gateway из ipconfig/ip route).
  2. Войдите под админом.
  3. Раздел Connected devices / DHCP clients / ARP — там имена хостов, локальные IP и MAC-адреса.

10) How to change your local IP (safe method)

  1. DHCP reservation: в админке роутера закрепите IP за MAC (лучше, чем статический IP на устройстве).
  2. Static IP: задайте IP из той же подсети, избегая DHCP-пула (например, пул 192.168.1.100–199 → выберите 192.168.1.210).
  3. Проверьте: ping gateway и доступ к ресурсам LAN.

13) Case example: printer offline due to IP conflict

В небольшом офисе принтер периодически «пропадал». На ПК команда arp -a показала две разные MAC-записи на 192.168.1.50: принтер и камера имели одинаковый статический IP. Решение: перевести оба устройства на DHCP и создать DHCP reservation для принтера. Конфликт исчез мгновенно.

14) Local vs Public IP — key differences

  • Local IP: виден только в LAN, выдается DHCP, используется для взаимодействия устройств.
  • Public IP: выдается ISP и виден сайтам/сервисам. Проверить: What is My IP.

15) Security & privacy tips

  • WPA2/WPA3 + уникальный admin пароль на роутере.
  • Guest-сеть — изолируйте от основной LAN.
  • Обновляйте прошивку роутера; периодически перезагружайте для очистки глючного DHCP-состояния.
  • Сделайте DHCP reservations для важных устройств (NAS/принтер).
  • Для внешней приватности используйте VPN (меняет public IP, но не local).
